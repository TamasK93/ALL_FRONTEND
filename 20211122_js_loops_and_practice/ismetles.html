String (karakter lanc)

let valtozo_neve = "valtozo erteke";
  • Nincs Char
  • String(), vagy toString()
  • van hossza, indexelheto
  • concatenation operator (+)
  • hasznos metodusai: includes, indexOf, slice, trim, charCodeAt

Number (lebegopontos szam)

let age = 33;
  • Number vagy parseInt()
  • Nincs hossza, nem indexelheto
  • NaN, isNaN
  • Number.MAX_SAFE_INTEGER, Number.MIN_SAFE_INTEGER
  • Infinity

Boolean

let isItTrue = true;
  • Boolean()
  • 0, NaN, undefined, null, "" == false

Egyenlo, vagy egyenlo es azonos tipusu 

if ([]) { console.log("[] is truthy")}         // logs "[] is truthy"
if ([] == false) { console.log("[] == false")} // logs "[] == false"

Az ok amiert [] == false hiaba true az ures tomb []: az osszehasonlitas [] == false az [] erteket hasonlitja a false-hoz. Ahhoz hogy az [] erteket megkapja a JS fordito meghivja a [].toString()-et. Ez egy ures string lesz "" es ezt fogja a false-hoz hasonlitani. Tehat [] == false az a JS ertelmezo szamara ugyan az, mint a "" == false. "" pedig falsy.
